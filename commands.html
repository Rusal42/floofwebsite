Automod Admin

View

%config automod view

Show current automod settings.

Enable/Disable Modules

%config automod enable <module>

Enable an automod module.

%config automod disable <module>

Disable an automod module.

Modules: spam, links, invites, caps, mentions, badwords, similar spam

These are the protections you can turn on/off.

Spam Settings

%config automod set spam max <n>

Max messages in time window.

%config automod set spam window <seconds>

Time window in seconds.

%config automod set spam mutetime <minutes>

Dynamic mute base (backward compatible).

%config automod set spam mutebase <minutes>

Set base mute duration.

%config automod set spam mutestep <minutes>

Increment per violation.

%config automod set spam mutemax <minutes>

Maximum mute duration.

Quick start: try max 6, window 8, mutebase 5.

Similar Message Spam

%config automod enable similarspam

Enable similar message spam checks.

%config automod disable similarspam

Disable similar message spam checks.

%config automod set similarspam minrepeats <n>

Messages in a row to trigger.

%config automod set similarspam action <delete|warn|mute>

Action on trigger.

%config automod set similarspam mutetime <minutes>

Mute duration when action=mute.

Tip: a good start is minrepeats 3, action mute, mutetime 10.

Links / Invites / Mentions

%config automod set links action <delete|warn|mute>

What to do when someone posts a link (non‚Äëwhitelisted).

%config automod set invites action <delete|warn|mute>

What to do when someone posts a server invite.

%config automod set mentions max <n>

How many @mentions are allowed in one message.

%config automod set mentions action <delete|warn|mute>

What to do if they go over the limit.

Actions: delete (remove), warn (message), mute (timeout).

%config automod links whitelist list

See who‚Äôs allowed to post links.

%config automod links whitelist clear

Admin Cheat Sheet 
# Aliases (modules)
# spam(s), links(link), invites(inv, invite), mentions(mention), badwords(bw, bad), similarspam(sim, simspam, similar)

# Aliases (keys)
# window(win), mutetime(mt), mutebase(mb), mutestep(ms), mutemax(mm), threshold(th), action(act), minrepeats(repeats|min)

# Examples using aliases
%config automod set spam win 8
%config automod set spam mb 5
%config automod set spam ms 5
%config automod set spam mm 60
%config automod set caps th 0.7
%config automod set sim act mute

%config automod disable link

All Automod settings are per-guild and require Administrator. Links module blocks all URLs from non-whitelisted users when enabled.

Menus

Rules / Agree Menu Admin

Use %config rulesmenu [#channel] to post. Members click the button to agree; optional auto-role if configured via %config rulescontent role:"RoleName".

  • Bot needs: Send Messages, Embed Links, Add Reactions, Manage Roles (if auto-role).
  • Common errors: role hierarchy (bot must be above the role), missing permissions.

Color Roles Menu Admin

Use %config colormenu [#channel] to post a color selector. Members pick a color; bot assigns the corresponding role. If color roles don‚Äôt exist yet, the bot will create the set automatically.

  • Bot needs Manage Roles, and its top role must be above the color roles.
  • Roles are created once on first run; subsequent runs reuse them.

Role Menu Admin

Use %config rolemenu <label> [#channel] <role1 role2 ...> to post a role selector. The embed title, description, and select placeholder are derived from <label> (e.g., label Games ‚Üí title ‚Äúüêæ Pick your Games!‚Äù).

  • Channel: if omitted, uses the configured roles channel or current channel.
  • Bot needs Manage Roles, Send Messages, and Embed Links.
  • Roles can be mentioned or named; missing roles may be created by the bot if supported.

Fun Roles Menu Admin

Use %config funroles [#channel] [role1 role2 ...] to post playful role menus (e.g., pings, notifications). If you omit role names, a default set is used.

  • Channel: if omitted, uses the configured roles channel or current channel.
  • Bot needs Manage Roles, Send Messages, and Embed Links.

Custom Rules Content Admin

Customize the rules embed text, footer, button label, and assign role via quoted key-value pairs. Newlines: use \n inside quotes.

%config rulescontent title:"Server Rules"
description:"1. Be respectful\n2. No spam\n3. Follow Discord ToS"
footer:"Click I Agree to get access" button:"I Agree" role:"Member"
  • Set defaults: %config ruleschannel #rules. Post using: %config rulesmenu.
  • Update any single field at any time (e.g., just the footer or button label).
  • Reset content: %config reset rulescontent
